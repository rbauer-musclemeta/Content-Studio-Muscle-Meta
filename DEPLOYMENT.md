This guide will help you deploy your Muscle-Meta Matrix platform to **Render.com** with **zero terminal commands** needed!

## 🎯 Deployment Overview

We'll deploy **3 components**:
1. **Backend (FastAPI)** → Render.com
2. **Frontend (React)** → Render.com
3. **Database (MongoDB)** → MongoDB Atlas

The `render.yaml` file in your repo will **automatically deploy both backend and frontend** in one click!

## STEP 1: Setup MongoDB Database (10 minutes)

### 1.1 Create MongoDB Atlas Account

1. Go to: **https://www.mongodb.com/cloud/atlas/register**
2. Click **"Sign up with Google"** (easiest) or create account
3. Complete the welcome survey (choose "Learning MongoDB" and "Node.js")

### 1.2 Create Free Cluster

1. Click **"Build a Database"**
2. Choose **"M0 FREE"** tier
3. Select **Cloud Provider**: AWS
4. Select **Region**: Choose closest to you (e.g., "us-west-2" for Oregon)
5. **Cluster Name**: Leave as "Cluster0" or rename to "MuscleMeta"
6. Click **"Create Cluster"** (takes 3-5 minutes)

### 1.3 Create Database User

1. On the Security Quickstart page, or go to **Security → Database Access**
2. Click **"Add New Database User"**
3. **Authentication Method**: Password
4. **Username**: `admin` (or your choice)
5. **Password**: Click "Autogenerate Secure Password" → **COPY AND SAVE IT!**
   - Or create your own strong password
6. **Database User Privileges**: Select "Read and write to any database"
7. Click **"Add User"**

### 1.4 Whitelist IP Addresses

1. Go to **Security → Network Access**
2. Click **"Add IP Address"**
3. Click **"Allow Access from Anywhere"** (adds 0.0.0.0/0)
   - This is needed for Render to connect
4. Click **"Confirm"**

### 1.5 Get Connection String

1. Go back to **Database** (left sidebar)
2. Click **"Connect"** button on your cluster
3. Choose **"Connect your application"**
4. **Driver**: Python, **Version**: 3.11 or later
5. **Copy the connection string** - looks like:
   ```
   mongodb+srv://admin:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```
6. **Replace `<password>`** with your actual database user password
7. **Save this connection string** - you'll need it in Step 3!

**Example:**
```
mongodb+srv://admin:MySecurePass123@cluster0.abc123.mongodb.net/?retryWrites=true&w=majority
```

✅ **MongoDB Setup Complete!**

## STEP 2: Setup Render Account (5 minutes)

### 2.1 Create Render Account

1. Go to: **https://render.com**

3. **Sign up with GitHub** (recommended - easiest integration)
4. Authorize Render to access your GitHub account

### 2.2 Connect Your GitHub Repository

1. You'll be prompted to "Install Render on GitHub"
2. Click **"Configure"**
3. Choose **"Only select repositories"**
4. Select **`muscle-metacourse-newsletter`**
5. Click **"Install"**

✅ **Render Setup Complete!**

## STEP 3: Deploy to Render (10 minutes)

### 3.1 Deploy Using render.yaml (One-Click Deploy!)

1. In Render dashboard, click **"New +"** → **"Blueprint"**
2. Select repository: **`muscle-metacourse-newsletter`**
3. **Branch**: `claude/phase-2-enhancement-planning-011CUmPBNrMcjZTkHyfQ5F8D`
4. Render will automatically detect `render.yaml`
5. Click **"Apply"**

Render will now create **2 services automatically**:
- `muscle-meta-backend` (FastAPI)
- `muscle-meta-frontend` (React)

### 3.2 Configure Environment Variables

You need to set **2 secret environment variables** that can't be in the config file:

#### For Backend Service:

1. Go to **Dashboard** → Click **`muscle-meta-backend`**
2. Click **"Environment"** tab (left sidebar)
3. Add these variables:

**MONGO_URL:**
```
mongodb+srv://admin:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```
(Use the connection string from Step 1.5)

**STRIPE_API_KEY:**
```
sk_test_YOUR_STRIPE_SECRET_KEY
```
(Get from: https://dashboard.stripe.com/test/apikeys)

4. Click **"Save Changes"**

**Note:** `JWT_SECRET_KEY` is auto-generated by Render ✅
**Note:** `REACT_APP_BACKEND_URL` is auto-filled from backend URL ✅

### 3.3 Wait for Deployment (5-10 minutes)

Both services will now build and deploy:
- **Backend**: Installing Python dependencies, starting FastAPI
- **Frontend**: Installing npm packages, building React app

**Watch the logs:**
- Click on each service name
- Click **"Logs"** tab
- You'll see build progress in real-time

**When successful, you'll see:**
- Backend: `Application startup complete`
- Frontend: `Accepting connections at http://0.0.0.0:XXXX`

### 3.4 Get Your Live URLs

Once deployed (both services show green "Live" badge):

1. Click **`muscle-meta-backend`**
   - Copy the URL (top of page): `https://muscle-meta-backend.onrender.com`

2. Click **`muscle-meta-frontend`**
   - Copy the URL: `https://muscle-meta-frontend.onrender.com`

**🎉 Your app is LIVE!**

## 🐛 Troubleshooting

### Issue: "Failed to fetch questions" on onboarding

**Cause:** Backend URL not configured correctly

**Fix:**
1. Go to frontend service → Environment
2. Check `REACT_APP_BACKEND_URL` equals your backend URL
3. Should NOT have trailing slash
4. Example: `https://muscle-meta-backend.onrender.com` ✅
5. Wrong: `https://muscle-meta-backend.onrender.com/` ❌

### Issue: "Could not connect to MongoDB"

**Cause:** MongoDB connection string incorrect or IP not whitelisted

**Fix:**
1. Check `MONGO_URL` has correct password
2. Verify MongoDB Atlas → Network Access has `0.0.0.0/0`
3. Test connection string by pasting in backend logs

### Issue: "Stripe API key not found"

**Cause:** Stripe API key not set

**Fix:**
1. Get key from: https://dashboard.stripe.com/test/apikeys
2. Use **Secret key** (starts with `sk_test_`)
3. Add to backend service environment variables

### Issue: Backend service "sleeping" or slow to wake

**Cause:** Render free tier puts inactive services to sleep after 15 min

**Fix (Optional):**
1. Sign up for free **UptimeRobot**: https://uptimerobot.com
2. Add monitor for your backend URL
3. Set to ping every 5 minutes
4. Keeps service awake 24/7

### Issue: Build failed - Python dependencies

**Cause:** Missing or incompatible dependencies

**Fix:**
1. Check Render logs for specific error
2. In `backend/requirements.txt`, ensure all packages have versions
3. Try pinning problematic packages to specific versions

### Issue: Frontend build failed - npm errors

**Cause:** Node modules issue

**Fix:**
1. Check if `frontend/package.json` is valid JSON
2. Verify `recharts` is listed in dependencies
3. Try clearing cache: In Render settings → "Clear build cache & deploy"

### Issue: CORS errors when frontend calls backend

**Cause:** CORS not configured for frontend domain

**Fix:**
1. This should be auto-handled by our CORS config
2. Verify `backend/server.py` has `allow_origins=["*"]`
3. Or whitelist specific origin in production

## 🔄 Updating Your Deployment

### Auto-Deploy on Git Push (Recommended)

Render automatically deploys when you push to GitHub:

1. Make changes locally

## 💰 Pricing & Limits

### Render Free Tier

**Included:**
- 750 hours/month (enough for 2 services running 24/7)
- Automatic SSL certificates
- Auto-deploys from GitHub
- Custom domains
- 100 GB bandwidth/month

**Limitations:**
- Services "sleep" after 15 min inactivity (wake in ~30 sec)
- Slower build times
- Less compute resources

**Upgrade to Paid ($7/mo per service):**
- No sleeping
- Faster builds
- More resources
- Priority support

### MongoDB Atlas Free Tier

**Included:**
- 512 MB storage
- Shared CPU/RAM
- Great for development/small apps

**Enough for:**
- ~50,000 user assessments
- ~10,000 course enrollments
